## 一、问题背景

云端 AstrBot 的 Alice Memory 插件定时任务全部失败，报错：

```
Context.llm_generate() missing 1 required keyword-only argument: 'chat_provider_id'
```

影响 4 个定时任务：01:00 Path B 日压缩、02:00 L1 清理、03:00 L3 衰减、04:00 Path A 周压缩。

### 根因

1. **代码缺陷**：`Compressor._call_llm()` 和 `Analyzer._call_llm()` 在 `umo=""`（定时任务场景）时未传递 `chat_provider_id` 参数
2. **部署陷阱**：本地 scp 上传时连带上传了 `__pycache__/*.cpython-313.pyc`（本地 Python 3.13），服务器 Python 3.12 优先加载旧字节码，导致源码更新后仍执行旧逻辑

## 七、已知注意事项

1. **scp 上传必须排除 `__pycache__`**：本地 Python 3.13 的 `.pyc` 在服务器 Python 3.12 上会导致加载旧代码
2. **`persistent=True` 会导致重启累积任务**：每次重启注册新任务，旧的不自动清理。当前已手动清理，后续版本应加入启动时去重逻辑
3. **AstrBot 数据目录与插件目录不在同一路径**：插件在 `/root/AstrBot/data/plugins/`，数据在 `/root/data/plugin_data/`
4. **数据库是 `data_v4.db`**，不是 `astrbot.db`
5. **错误日志 `astrbot.err` 中的 `coroutine was never awaited`** 是旧 `.pyc` 缓存遗留，当前代码已修复
